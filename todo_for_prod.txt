# TODO LIST : PASSAGE EN PRODUCTION (POP'IVOIRE)

## 🚨 URGENT / INDISPENSABLE (Avant mise en ligne)

### 1. Configuration Serveur & Sécurité
- [ ] **Variables d'environnement (.env)** :
  - Sur le serveur, créer le fichier `.env` à partir de `.env.production.example`.
  - Générer une vraie `SECRET_KEY` (longue et aléatoire).
  - Mettre `DEBUG=False`.
  - Remplir `ALLOWED_HOSTS` avec vos domaines (popivoire.net, www.popivoire.net).
- [ ] **Base de Données (PostgreSQL)** :
  - Installer PostgreSQL sur le serveur.
  - Créer la base `popivoire_db` et l'utilisateur `popivoire_admin`.
  - Mettre à jour les identifiants DB dans le fichier `.env` du serveur.
- [ ] **Fichiers Statiques** :
  - Exécuter `python manage.py collectstatic` sur le serveur pour rassembler tous les fichiers CSS/JS.
  - Vérifier que WhiteNoise sert bien les fichiers (automatique avec ma config, mais à tester).

### 2. Service Email (SMTP)
- [ ] **Fournisseur Email** :
  - Créer un compte sur un service d'envoi transactionnel (SendGrid, Mailgun, Brevo/Sendinblue) ou utiliser un compte Gmail dédié (avec mot de passe d'application).
- [ ] **Configuration** :
  - Remplir les variables `EMAIL_HOST`, `EMAIL_PORT`, `EMAIL_HOST_USER`, `EMAIL_PASSWORD` dans le `.env`.
- [ ] **Test** :
  - S'inscrire à la newsletter ou créer un compte pour vérifier la réception des emails.

### 3. Hébergement & Domaine
- [ ] **DNS** :
  - Faire pointer le domaine `popivoire.net` (Type A) vers l'adresse IP publique du serveur VPS.
- [ ] **Serveur Web (Nginx + Gunicorn)** :
  - Installer et configurer Gunicorn pour exécuter Django.
  - Installer et configurer Nginx comme "Reverse Proxy" devant Gunicorn.
  - **IMPORTANT** : Configurer Nginx pour servir le dossier `media/` (les images uploadées par les utilisateurs), car Django ne le fait pas en production (contrairement aux fichiers statiques gérés par WhiteNoise).
- [ ] **HTTPS (SSL)** :
  - Installer `certbot` et générer un certificat Let's Encrypt pour sécuriser le site (cadenas vert).
